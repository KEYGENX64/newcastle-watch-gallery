# معرفی

بک‌اند این پروژه با **Laravel** پیاده‌سازی شده و نقش هسته‌ی اصلی سیستم را بر عهده دارد.
تمامی درخواست‌های مربوط به **ورود کاربران، مدیریت محصولات، دسته‌بندی‌ها، کاربران و پروفایل** در این بخش پردازش می‌شوند و خروجی نهایی به صورت **JSON** در اختیار فرانت‌اند قرار می‌گیرد.

---

# مسئولیت‌های بک‌اند

این بخش مسئول موارد زیر است:

* پیاده‌سازی APIهای پروژه
* مدیریت احراز هویت با **JWT**
* اعتبارسنجی داده‌های ورودی
* مدیریت کاربران و نقش‌ها
* مدیریت محصولات و دسته‌بندی‌ها
* ارتباط با پایگاه داده MySQL
* تولید خروجی استاندارد برای فرانت‌اند
* کنترل سطح دسترسی کاربران از طریق Middleware

# راه‌اندازی پروژه

## پیش‌نیازها

قبل از اجرای پروژه، ابزارهای زیر باید نصب باشند:

* PHP
* Composer
* MySQL
* WampServer یا هر محیط محلی مشابه برای اجرای Apache/MySQL

---

## نصب وابستگی‌ها

در پوشه `backend` دستور زیر را اجرا کنید:

```bash id="be-1"
composer install
```

---

## فایل تنظیمات `.env`

فایل `.env` باید بر اساس تنظیمات محیط محلی شما تنظیم شود.
در صورتی که فایل `.env` در مخزن وجود نداشته باشد، می‌توانید از فایل نمونه استفاده کنید:

```bash id="be-2"
cp .env.example .env
```

مقادیر اصلی که باید بررسی یا تنظیم شوند:

* `DB_DATABASE`
* `DB_USERNAME`
* `DB_PASSWORD`

---

## تولید کلید برنامه

```bash id="be-3"
php artisan key:generate
```

---

## اجرای Migration و Seeder

برای ساخت جداول پایگاه داده و ورود اطلاعات اولیه:

```bash id="be-4"
php artisan migrate --seed
```

این دستور موارد زیر را انجام می‌دهد:

* ایجاد جداول پایگاه داده
* ثبت دسته‌بندی‌های اولیه
* ثبت محصولات اولیه
* ساخت کاربران پیش‌فرض سیستم

---

## اجرای سرور توسعه

```bash id="be-5"
php artisan serve
```

آدرس پیش‌فرض اجرای بک‌اند:

```text id="be-6"
http://127.0.0.1:8000
```

آدرس پایه APIها:

```text id="be-7"
http://127.0.0.1:8000/api
```

# تنظیمات محیط

نمونه‌ای از تنظیمات موردنیاز در فایل `.env`:

```env id="be-8"
APP_NAME=Newcastle
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://127.0.0.1:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

# احراز هویت

در این پروژه از **JWT** برای احراز هویت استفاده شده است.

## روند کلی

1. کاربر از طریق فرم ورود، نام کاربری و رمز عبور را ارسال می‌کند.
2. در صورت معتبر بودن اطلاعات، توکن JWT تولید می‌شود.
3. توکن در فرانت‌اند ذخیره شده و در درخواست‌های بعدی ارسال می‌شود.
4. Middleware مربوط به JWT توکن را اعتبارسنجی کرده و کاربر را شناسایی می‌کند.

# نکات مهم

* فایل `.env` را داخل مخزن commit نکنید.
* برای انتشار مخزن، فایل `.env.example` را نگه دارید.
* پوشه `vendor/` نباید در Git نگهداری شود.
* برای اجرای صحیح فرانت‌اند، آدرس بک‌اند باید با مقدار `server` در فایل `frontend/src/environments/environment.ts` هماهنگ باشد.

# جمع‌بندی

پوشه `backend` هسته‌ی اصلی پروژه است و مسئول منطق کسب‌وکار، احراز هویت، کنترل دسترسی و ارتباط با پایگاه داده می‌باشد.
این بخش یک **Laravel API** کامل برای مدیریت **کاربران، دسته‌بندی‌ها و محصولات** فراهم می‌کند و به عنوان منبع داده برای فرانت‌اند Angular عمل می‌کند.
