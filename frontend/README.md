# معرفی

فرانت‌اند این پروژه با **Angular** پیاده‌سازی شده و به عنوان رابط کاربری سیستم عمل می‌کند.
این بخش با استفاده از APIهای Laravel داده‌ها را دریافت و نمایش می‌دهد و عملیات مختلفی مانند **ورود، مشاهده و مدیریت محصولات، دسته‌بندی‌ها، کاربران و پروفایل** را در اختیار کاربر قرار می‌دهد.

---

# مسئولیت‌های فرانت‌اند

این بخش مسئول موارد زیر است:

* نمایش صفحات و رابط کاربری سیستم
* مدیریت فرم‌های ورود، افزودن، ویرایش و جزئیات
* برقراری ارتباط با APIهای بک‌اند
* مدیریت وضعیت ورود کاربر
* ذخیره و استفاده از توکن JWT
* محافظت از مسیرهای خصوصی
* تفکیک دسترسی کاربر عادی و مدیر در رابط کاربری

# راه‌اندازی پروژه

## پیش‌نیازها

قبل از اجرای پروژه، ابزارهای زیر باید نصب باشند:

* Node.js نسخه `18` یا بالاتر
* npm
* Angular CLI

برای نصب Angular CLI:

```bash id="fe-1"
npm install -g @angular/cli
```

---

## نصب وابستگی‌ها

در پوشه `frontend` دستور زیر را اجرا کنید:

```bash id="fe-2"
npm install
```

---

## اجرای پروژه

```bash id="fe-3"
npm start
```

یا در برخی تنظیمات Angular:

```bash id="fe-4"
ng serve
```

آدرس پیش‌فرض اجرای فرانت‌اند:

```text id="fe-5"
http://localhost:4200
```

---

# تنظیمات اتصال به بک‌اند

آدرس سرور بک‌اند در فایل زیر تنظیم می‌شود:

```text id="fe-6"
src/environments/environment.ts
```

نمونه ساختار:

```ts id="fe-7"
export const environment = {
  production: false,
  model: "development",
  server: "http://127.0.0.1:8000",
  offsetSec: 0
}
```

> مقدار `server` باید با آدرس اجرای Laravel هماهنگ باشد.

# بخش‌های اصلی سیستم

## 1) لایه API

در مسیر `src/app/core/api` یک سرویس مرکزی برای ارسال درخواست‌ها به بک‌اند وجود دارد.

### مسئولیت‌های این لایه

* ارسال درخواست‌های `GET`
* ارسال درخواست‌های `POST`
* ارسال درخواست‌های `PUT`
* ارسال درخواست‌های `DELETE`
* ارسال فرم همراه فایل
* مدیریت پارامترهای درخواست

---

## 2) Repositoryها

برای هر موجودیت، یک Repository مستقل تعریف شده است تا صفحات به صورت مستقیم با `ApiService` درگیر نباشند.

### Repositoryهای اصلی

* `ProductRepository`
* `CategoryRepository`
* `UserRepository`

### مزایای این ساختار

* کاهش وابستگی مستقیم صفحات به API
* خواناتر شدن کد
* ساده‌تر شدن نگهداری و توسعه پروژه
* تبدیل پاسخ‌های API به مدل‌های قابل استفاده در UI

---

## 3) احراز هویت

فرانت‌اند دارای سرویس اختصاصی برای احراز هویت است که وظیفه‌ی مدیریت ورود و خروج کاربران را بر عهده دارد.

### قابلیت‌ها

* ارسال درخواست ورود
* ذخیره توکن JWT در `LocalStorage`
* بررسی اعتبار توکن
* خروج از سیستم
* بازیابی اطلاعات کاربر از روی توکن

---

## 4) مسیرها و محافظت از صفحات

برای کنترل دسترسی به صفحات از Guard استفاده شده است.

### Guardهای اصلی

* `AuthGuard`
  جلوگیری از دسترسی کاربران مهمان به صفحات محافظت‌شده

* `NoAuthGuard`
  جلوگیری از ورود کاربران لاگین‌شده به صفحه ورود

# بخش‌های اصلی سیستم

## ماژول احراز هویت

مسیرهای مربوط به ورود و خروج کاربران در این بخش قرار دارند:

* `sign-in`
* `sign-out`

### صفحات اصلی

* صفحه ورود
* صفحه خروج

---

## ماژول مدیریت / پنل اصلی

بخش مدیریت به عنوان Container صفحات اصلی سیستم عمل می‌کند و از طریق `RouterOutlet` صفحات مختلف را نمایش می‌دهد.

### نقش‌ها

* **مدیر سیستم**: دسترسی به کاربران، دسته‌بندی‌ها و محصولات
* **کاربر عادی**: دسترسی به محصولات، دسته‌بندی‌ها و پروفایل شخصی

---

## ماژول دسته‌بندی‌ها

این بخش شامل صفحات زیر است:

* لیست دسته‌بندی‌ها
* افزودن دسته‌بندی
* ویرایش دسته‌بندی

### مسیرهای اصلی

* `/category`
* `/category/add`
* `/category/edit/:id`

---

## ماژول کاربران

این بخش شامل صفحات زیر است:

* لیست کاربران
* افزودن کاربر
* ویرایش کاربر
* پروفایل
* تغییر رمز عبور کاربر جاری
* تغییر رمز عبور سایر کاربران توسط مدیر

### مسیرهای اصلی

* `/user`
* `/user/add`
* `/user/edit/:id`
* `/user/profile`
* `/user/change-my-password`

---

## ماژول محصولات

این بخش شامل صفحات زیر است:

* لیست محصولات
* افزودن محصول
* ویرایش محصول
* جزئیات محصول
* کارت نمایش محصول

### مسیرهای اصلی

* `/product`
* `/product/add`
* `/product/edit/:id`
* `/product/detail/:id`

---

# احراز هویت در فرانت‌اند

فرآیند کلی احراز هویت در فرانت‌اند به این صورت است:

1. کاربر نام کاربری و رمز عبور را در فرم ورود وارد می‌کند.
2. درخواست ورود به API بک‌اند ارسال می‌شود.
3. در صورت موفقیت، توکن JWT در `LocalStorage` ذخیره می‌شود.
4. اطلاعات کاربر از روی JWT استخراج می‌شود.
5. مسیرهای محافظت‌شده فقط برای کاربران معتبر در دسترس قرار می‌گیرند.

# نکات مهم

* قبل از اجرای فرانت‌اند، مطمئن شوید بک‌اند Laravel در حال اجرا است.
* مقدار `server` در فایل `environment.ts` باید به آدرس صحیح بک‌اند اشاره کند.
* پوشه `node_modules/` نباید در مخزن Git نگهداری شود.
* خروج و ورود کاربران بر پایه‌ی JWT و نگهداری توکن در مرورگر انجام می‌شود.

# جمع‌بندی

پوشه `frontend` شامل رابط کاربری پروژه است و با استفاده از **Angular**، صفحات مربوط به **ورود، مدیریت کاربران، مدیریت دسته‌بندی‌ها، مدیریت محصولات و پروفایل** را پیاده‌سازی می‌کند.
این بخش از طریق یک ساختار **Repository + ApiService** به بک‌اند Laravel متصل می‌شود و نقش نمایش و تعامل کاربر با سیستم را بر عهده دارد.
